# Task001-项目启动记录（企业微信账号体系复用）

## 1. 会议信息

- 项目：工单系统企业微信账号体系复用（一期）
- 会议类型：项目启动会
- 会议日期：2026-03-04
- 会议目标：统一一期范围、约束条件、验收口径与执行节奏

## 2. 一期范围确认

### 2.1 In Scope（确认纳入）

1. 企微配置管理：`corpId`、`agentId`、`corpSecret`
2. 部门/员工同步：手动同步 + 定时同步
3. 组织查询：部门树、员工分页、员工详情
4. 认证登录：企微扫码登录与会话管理
5. 工单联动：自动派单、升级派单、离职失权
6. 稳定性保障：同步日志、监控指标、告警、回滚机制

### 2.2 Out of Scope（确认不纳入）

1. 多租户隔离
2. 跨企业通讯录聚合
3. 复杂审批流改造
4. 在工单系统内直接维护企微主数据

## 3. 关键约束与前置条件

1. 企业微信为唯一身份源、唯一组织源，工单系统仅持久化副本。
2. 测试与上线必须使用真实企微账号与真实网络环境（禁止mock）。
3. 认证与授权分离：企微负责认证，工单系统本地RBAC负责授权。
4. 同步必须满足幂等、可重试、可审计，不得产生重复脏数据。

## 4. 环境与账号基线检查

### 4.1 环境检查清单

- [ ] 测试环境可访问企业微信开放接口
- [ ] `corpId`、`agentId`、`corpSecret` 已注入测试环境
- [ ] 测试库已具备组织与同步日志相关表结构
- [ ] 日志采集链路已打通（应用日志 + 同步日志）

### 4.2 真实账号检查清单

- [ ] 普通员工账号 >= 2
- [ ] 部门主管账号 >= 1
- [ ] 停用或离职账号 >= 1（用于失权验证）
- [ ] 至少一名跨部门员工（用于主部门映射验证）

## 5. 指标口径（启动基线）

1. 同步成功率：`>= 99%`
2. 登录成功率：`>= 95%`
3. 自动派单命中率：`>= 95%`
4. P0/P1测试用例通过率：`100%`

## 6. 里程碑与时间线（10个工作日）

1. M1（D2）：数据模型与企微连接可用
2. M2（D5）：同步闭环与组织查询可用
3. M3（D7）：登录会话与工单联动可用
4. M4（D10）：联调验收通过并具备上线条件

## 8. 风险与值班机制

### 8.1 高风险场景

1. 同步连续失败导致组织数据过期
2. 扫码登录不可用导致用户无法登录
3. 离职用户权限未即时回收导致越权
4. 派单规则异常导致工单错误分配

### 8.2 值班机制（上线窗口）

1. 值班时段：发布日及后续24小时重点观察
2. 升级路径：测试/业务 -> 研发值班 -> 运维值班 -> 负责人
3. 响应时限：P0 15分钟内响应，P1 30分钟内响应

## 9. 回滚触发条件（启动会确认）

1. 扫码登录不可用持续超过15分钟
2. 同步任务连续失败超过3次
3. 出现离职用户可操作工单等大面积权限异常
4. 自动派单错误率持续高于20%

## 10. 启动会结论

1. 一期范围和不做项已确认，不新增越界需求。
2. 按“先基座后闭环”推进：先完成模型与连接，再推进同步、查询、认证、派单。
3. 回滚阈值与值班机制按本记录执行，上线前需完成一次演练。
