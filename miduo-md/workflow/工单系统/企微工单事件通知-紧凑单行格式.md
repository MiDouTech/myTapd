# 企微工单事件通知紧凑单行格式

## 背景

企业微信群「工单事件通知」原为多段分区排版（事件、时间、工单信息、变更内容等），信息密度高但扫读成本高。现统一改为单行紧凑格式，便于在群里快速识别工单与跳转详情。

## 目标格式

```text
【工单编号】【状态】【标题】详情链接
```

示例：

```text
【WO-20260617-013-9441】【已完成】【奖品订单中有部分订单超过所设定的自动确认收货时间，却一直没有自动确认收货】https://ticket.ebcone.cn/open/ticket/WO-20260617-013-9441
```

## 影响范围

| 通道 | 实现 |
|------|------|
| Webhook 企微机器人 text | `WebhookDispatchService#buildWecomRobotTextPayload` |
| 企微群绑定推送 | `WecomGroupPushService` + `TicketWecomCompactNotificationBuilder` |
| 评论@提醒 | 保持原多段格式（需展示评论人与摘要） |
| Bug 简报归档 | 保持原格式 |

## 非目标

- 不改变站内信、企微应用消息等个人通知的正文结构。
- 不改变 Webhook JSON 原始负载，仅调整企微机器人 text 适配层。

## 验收

1. 触发工单创建/分派/状态变更/关闭，群消息为单行紧凑格式。
2. 状态展示为中文标签（如「已完成」「待受理」）。
3. 详情链接为 `ticket.detail-url/{ticketNo}`。
4. @mention 逻辑保持不变。
