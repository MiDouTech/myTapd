# 工作流详情可视化优化方案

> **业务模块**：工单系统 / 管理端工作流管理  
> **关联任务**：Task005、Task016、Task017  
> **版本**：v1.0  
> **日期**：2026-04-03

---

## 一、背景

当前 `/manage/workflow` 页面已经支持：

- 工作流列表查询
- 工作流详情抽屉查看
- 状态定义与流转规则维护

但详情页仍以「基础信息 + 状态表 + 流转规则表」为主，业务、产品、运营在查看时很难快速回答以下问题：

1. 流程从哪里开始、到哪里结束
2. 主路径和退回/驳回路径分别是什么
3. 哪些角色能执行某一步流转
4. 实际工单经常卡在哪个节点

这会导致工作流评审、流程培训、异常排查都需要额外口头解释或截图标注，系统本身无法承担“流程认知载体”的角色。

---

## 二、现状问题

### 2.1 认知方式偏表格，不偏流程

当前详情页数据完整，但需要用户自行在脑中拼接：

```text
状态 A
状态 B
状态 C

A -> B
B -> C
B -> A
```

复杂流程存在回退、挂起、驳回时，理解成本明显升高。

### 2.2 定义与运行视角割裂

工单详情页已经具备流转历史时间线，但工作流管理页无法直接查看：

- 最近真实流转轨迹
- 节点进入次数
- 平均停留时长
- 常见退回路径

流程“怎么设计”和“怎么运行”之间缺少闭环。

### 2.3 详情页主视图不符合用户目标

用户进入工作流详情页的首要目标通常是：

> 先看懂流程怎么走，再核对规则怎么配。

当前页面的默认主视图却是表格，导致与用户心智不一致。

---

## 三、优化目标

### 3.1 目标

1. 将工作流详情从“配置明细页”升级为“流程认知页”
2. 默认以图形化方式展示状态流转关系
3. 补充矩阵视图与规则明细视图，兼顾审查与核对
4. 引入轻量运行观察视图，展示真实流转和节点耗时概览

### 3.2 非目标

本次优化不包含：

- 拖拽式 BPMN 设计器
- 复杂自动化编排编辑器
- 多泳道审批建模
- 全量统计报表中心

本次聚焦“可读”“可解释”“可观察”。

---

## 四、方案概要

工作流详情页改造为四个主视图：

1. **流程图视图（默认）**
2. **流转矩阵视图**
3. **规则明细视图**
4. **运行观察视图**

页面整体结构：

```text
工作流详情
├─ 顶部概览区
├─ 流程图视图（默认）
├─ 流转矩阵视图
├─ 规则明细视图
└─ 运行观察视图
```

---

## 五、详细设计

### 5.1 顶部概览区

用于首屏快速建立认知，展示：

- 工作流名称
- 模式（SIMPLE / ADVANCED）
- 启用状态
- 状态数
- 流转数
- 初始状态
- 终态数量
- 退回流转数量
- 最近更新时间
- 描述

### 5.2 流程图视图

使用已有 `states` / `transitions` 数据构建状态机图：

- 节点：展示状态名称、状态类型、SLA 动作
- 边：展示流转关系，区分正常流转与退回流转
- 图例：初始 / 中间 / 终态、正常 / 退回 / 异常路径
- 属性面板：点击节点或边查看详细属性

设计要求：

- 初始节点使用主色高亮
- 终态节点使用终态样式强调
- 退回流转使用虚线
- 不在节点上堆叠过多字段，详细信息统一在右侧属性面板承接

### 5.3 流转矩阵视图

用二维矩阵辅助规则审查：

- 行：来源状态
- 列：目标状态
- 单元格：流转名称、角色摘要、退回标记

适用于：

- 管理员核对可达关系
- 产品/测试审查是否存在冗余流转
- 快速识别单向死路或回退过多问题

### 5.4 规则明细视图

保留当前表格能力，但从“默认主视图”降级为“辅助核对视图”：

- 状态定义表：状态编码、名称、类型、SLA 动作、排序
- 流转规则表：起始状态、目标状态、流转名称、角色、备注必填、可转派、退回

### 5.5 运行观察视图

基于真实运行数据提供轻量观察能力：

- 最近流转时间线
- 节点进入次数
- 平均停留时长
- 最大停留时长
- 平均等待/处理耗时

首版仅提供“轻量观察”，不扩展复杂统计报表。

## 六、接口与数据调整

### 6.1 复用接口

- `GET /api/workflow/detail/{id}`（API000201）
  - 补充状态排序、流转规则属性（ID、备注必填、可转派、退回）

### 6.2 新增接口

- `GET /api/workflow/observation/{id}`（API000206）
  - 返回工作流运行观察信息
  - 包含最近流转记录、节点耗时统计、工单数量等轻量数据

### 6.3 前端新增能力

- 工作流流程图组件
- 工作流矩阵组件
- 工作流运行观察面板

## 八、兼容与回滚策略

### 8.1 兼容策略

- 现有 `GET /api/workflow/detail/{id}` 保持原路径不变，仅补充字段
- 旧版详情页仍依赖的基础字段保持兼容
- 运行观察独立为新增接口，不影响已有列表/编辑能力

### 8.2 回滚策略

- 前端可回滚为旧版详情抽屉
- 新增 observation 接口可单独下线，不影响详情基础能力
- 详情增强字段为向后兼容扩展，不影响旧调用方
