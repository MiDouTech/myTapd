# 工单关闭不自动关联 Bug 简报

## 背景与问题

缺陷类工单进入任意终态时，系统会发布 `TicketCompletedEvent`，由 `BugReportEventListener` 调用 `createDraftFromClosedTicket`：若工单尚未关联简报，则**自动创建简报草稿并写入关联**。

用户选择**「已关闭」**（`closed`，如非缺陷关闭等路径）时，通常表示无需再走简报闭环，自动关联会产生多余草稿与通知。  
线上实际反馈中，工单走到「已完成」后又回退上一步，再次点击「已关闭」时，系统会残留此前自动生成的草稿简报，导致“已关闭仍有简报”。

## 目标

- 工单终态为 **`closed`** 时，**不再**自动创建 Bug 简报草稿、不自动写入工单-简报关联。
- 若工单此前已自动生成过草稿简报（例如“已完成 -> 回退 -> 已关闭”），关闭时应清理该自动草稿，确保简报列表不残留对应记录。
- 终态为 **`completed`** 等其他终态时，保持原有「无关联则自动建草稿」行为（不改变验收完成后的简报闭环）。

## 非目标

- 不修改用户手动在简报侧关联工单的流程。
- 不删除历史已自动生成的简报数据。

## 实现要点

- 在 `BugReportEventListener.onTicketCompleted` 中判断 `finalStatus`：
  - 若为 `closed`（忽略大小写），先调用自动草稿清理逻辑，再直接返回，不调用 `createDraftFromClosedTicket`；
  - 仅非 `closed` 的终态继续走自动创建/复用简报逻辑。

## 验收

- 缺陷工单直接进入 **已关闭**：详情页不应因本次关闭**新增**自动生成的简报关联（无历史关联时仍无关联）。
- 缺陷工单进入 **已完成**：若此前无关联简报，仍会生成简报草稿并关联（与优化前一致）。
- 缺陷工单执行 **已完成 -> 回退上一步 -> 已关闭**：关闭后不应保留对应自动草稿，Bug简报列表中该自动简报应消失。
