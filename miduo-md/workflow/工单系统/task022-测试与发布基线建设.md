# Task022：测试与发布基线建设

> **业务模块**：工单系统（质量专项）  
> **依赖**：Task016~Task021  
> **预估工时**：2~3天  
> **对应目标**：建立“可回归、可发布、可追溯”的工程质量底座

---

## 一、任务目标

补齐当前项目“能跑但缺少自动验证”的短板，建立前后端最小可行测试体系、联调回归清单与发布门禁，保障管理端新增功能稳定上线。

## 二、当前痛点

1. 前后端自动化测试缺失，回归依赖人工  
2. 发布前检查项不统一，容易漏测  
3. 文档与实际运行结果缺少闭环  
4. 环境差异（如 Maven 缺失）影响可验证性

## 三、建设范围

### 3.1 前端测试基线

- 引入单元测试框架（建议 Vitest + Vue Test Utils）
- 覆盖核心页面与组件最小冒烟：
  - 管理端四页可渲染
  - 通知中心关键交互
  - Bug简报列表与详情关键逻辑
  - API 错误态与空态处理

### 3.2 后端测试基线

- 增加关键接口集成测试（SpringBootTest/MockMvc）
- 覆盖高价值接口：
  - 分类、工单、工作流、SLA、通知、Webhook
- 增加基础鉴权场景：
  - 未登录 401
  - 参数非法校验失败

### 3.3 回归与发布清单

- 建立标准回归文档模板：
  - 功能回归
  - 接口回归
  - 性能与稳定性基础检查
- 发布前门禁项：
  - 前端 lint + build + test
  - 后端 compile + test
  - 接口编号校验脚本（Task021）

### 3.4 环境可重复验证

- 固化本地/云端执行命令
- 输出环境依赖说明（JDK/Maven/Node 版本）
- 提供常见失败场景排障指引

## 四、详细实施内容

### 4.1 前端自动化测试建议

- 目录：`miduo-frontend/src/__tests__/`
- 示例用例：
  - `manage-workflow-view.spec.ts`
  - `sla-manage-view.spec.ts`
  - `notification-center.spec.ts`
  - `bugreport-list.spec.ts`
- 关键断言：
  - 成功场景渲染
  - 失败场景提示
  - 空列表空态
  - 分页交互触发请求参数变化

### 4.2 后端自动化测试建议

- 目录：`ticket-platform/**/src/test/java`
- 示例用例：
  - `CategoryControllerTest`
  - `SlaPolicyControllerTest`
  - `NotificationControllerTest`
  - `WorkflowControllerTest`
  - `WebhookConfigControllerTest`
- 关键断言：
  - HTTP 状态与返回结构
  - 参数校验消息
  - 典型业务路径正确

### 4.3 发布门禁脚本建议

- 前端：
  - `npm run lint`
  - `npm run build`
  - `npm run test`（新增）
- 后端：
  - `mvn -pl ticket-bootstrap -am test`
- 治理：
  - `node scripts/qa/check-api-numbering.mjs`

## 六、实施步骤建议

### Day 1
- 补前端测试框架与关键页面冒烟用例
- 固化前端测试命令到 package scripts

### Day 2
- 补后端关键接口测试（优先管理端相关接口）
- 建立发布前检查清单

### Day 3（缓冲）
- 连续回归运行、修复波动用例
- 输出质量报告与遗留风险清单

## 七、验收标准

- [ ] 前后端均具备可执行的自动化测试命令  
- [ ] 新增管理端能力（Task016~020）具备最小冒烟覆盖  
- [ ] 发布前检查清单可执行并形成记录  
- [ ] 编号治理校验纳入发布流程  
- [ ] 质量报告可追溯，明确未覆盖风险  

## 八、风险与应对

- **风险1：短期补测成本高**  
  处理：先覆盖高风险主链路，再按缺陷密度逐步扩展。

- **风险2：测试依赖外部环境不稳定**  
  处理：将可控场景前置为本地可重复执行，外部依赖场景单独标记。

- **风险3：门禁过严影响迭代速度**  
  处理：分级门禁（必须项/建议项），先保住核心质量红线。

## 九、完成定义（DoD）

当以下条件全部满足时，Task022 视为完成：

1. 管理端新增功能均有自动化冒烟覆盖  
2. 发布前检查流程文档化并执行留痕  
3. 至少连续两次回归通过，无阻塞级缺陷  
4. 遗留风险清单已评审确认并有后续处理计划
