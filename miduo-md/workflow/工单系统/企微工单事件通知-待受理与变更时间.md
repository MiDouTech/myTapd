# 企微工单事件通知：待受理处理人与变更时间

## 背景与目标

企业微信群机器人「工单事件通知」在**工单状态变更**场景下，需更易读、可行动：
- 进入**待受理类**状态（待分派、待受理、待测试受理、待开发受理）时，应直接看到**当前处理人**。
- **变更内容**区块末尾补充**变更时间**，与事件时间对齐，避免仅顶部一行时间时扫读遗漏。

## 方案概要

| 项 | 说明 |
|----|------|
| 实现位置 | `ticket-platform` → `WebhookDispatchService#buildWecomRobotTextPayload` |
| 待受理判定 | `TicketStatus#isPendingAcceptanceLike()`（枚举层统一维护） |
| 当前处理人 | 工单快照 `assigneeId` → `SysUserMapper` 解析姓名；无则 `-` |
| 变更时间 | 存在「变更内容」时，在编号行之后追加 `变更时间：{eventTime}`，缺省用组装时刻 |

## 非目标

- 不改变非企微机器人 Webhook 的 JSON 负载结构。
- 不在无「变更内容」的事件中强行插入变更时间块。

## 验收

1. 将工单流转到「待开发受理」等待受理类状态，群消息「工单信息」含 **5) 当前处理人：xxx**。
2. 同一条消息「变更内容」最后有一行 **变更时间**，与顶部「【时间】」一致。
