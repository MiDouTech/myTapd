# 工单列表状态多选筛选（体验优化）

## 背景与问题

「我的工单 / 所有工单」列表的状态筛选为单选，无法一次查看多种流程状态下的工单组合。

## 目标

状态筛选支持多选；选中多个状态时，列表返回**任一**状态命中的工单（与 SQL `IN` 语义一致）。

## 非目标

- 不改变视图（我待办/我创建等）与关键词、SLA 等其它筛选项的语义。
- 不要求用户在 URL 中手写多状态（仪表盘跳转仍可用单个 `?status=`）。

## 方案概要

- **前端**：`el-select` 开启 `multiple`，请求参数使用 `statuses` 字符串数组；`axios` 对数组使用 `paramsSerializer: { indexes: null }` 生成 `statuses=a&statuses=b`，便于 Spring MVC 绑定为 `List<String>`。
- **后端**：`TicketPageInput` 增加 `statuses`；与原有单字段 `status` 在 Service 层合并去重后传入 Mapper；XML 中 `t.status IN (...)`。兼容仅传 `status` 的旧调用（含开放接口）。

## 影响范围

- 前端：`miduo-frontend` 工单列表页 `TicketListView.vue`、`TicketPageInput` 类型、`getTicketPage` 请求序列化。
- 后端：`ticket-platform` 中 `TicketPageInput`、`TicketMapper`/`TicketMapper.xml`、`TicketApplicationService#getTicketPage`。

## 验收方式

1. 在工单列表状态框勾选 2 个及以上不同状态，点击查询，列表中每条工单的「状态」属于所选集合之一。
2. 仅选一个状态时行为与原先单选一致。
3. 从仪表盘带 `?status=xxx` 进入列表时，该状态仍出现在筛选中并生效。
