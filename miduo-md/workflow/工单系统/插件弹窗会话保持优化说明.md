# 插件弹窗会话保持优化说明

> **日期**：2026-07-21  
> **类型**：体验优化（不改变对外 API 契约，仅调整默认配置与 SDK 行为）  
> **关联**：业务原生工单插件方案、task031

---

## 一、背景与问题

用户反馈：嵌入业务系统（如 BDE 大数据看板）的「提交工单」插件弹窗，**维持约 4 分钟后会自动退出**，填写中的描述与图片可能丢失。

## 二、现状分析

| 环节 | 现状 | 问题 |
|------|------|------|
| LaunchToken 有效期 | 默认 `300` 秒（5 分钟） | 用户从打开页面到点开弹窗、再填写截图与描述，很容易超过有效期 |
| SDK 弹窗关闭 | 仅用户点击 `×/取消`、提交成功 1.2s 后自动关闭 | SDK 本身**无** 4 分钟定时关闭逻辑 |
| 业务侧集成 | 部分系统按 `expireSeconds` 定时销毁插件或重新 `init` | Token 过期后弹窗被外层逻辑关掉，表现为「自动退出」 |

**类比**：像游乐园手腕票只有 5 分钟有效，你刚进园区还没玩到项目，票就作废被请出去了。

## 三、优化目标

1. 弹窗**不因 LaunchToken 到期而自动关闭**，仅用户手动点击或提交成功后关闭。
2. Token 有效期覆盖「打开弹窗 → 写描述 → 上传截图 → 提交」的完整路径。
3. 业务侧可在 Token 将过期时调用 `TicketSDK.refreshLaunchToken()` 静默续期，无需销毁弹窗。

## 四、非目标

- 不改动插件建单、上传附件等 API 路径与鉴权模型。
- 不强制业务系统改造集成方式（仅提供续期能力与文档指引）。

## 五、方案概要

### 5.1 后端：延长 LaunchToken 默认有效期

- `plugin.launch-token-expire-seconds` 默认值由 `300` 调整为 `7200`（2 小时）。
- 仍可通过 Nacos / 环境变量覆盖。

### 5.2 SDK：会话保持与续期

- 新增 `TicketSDK.refreshLaunchToken(launchToken)`，业务后端重新签发后注入，**不关闭已打开弹窗**。
- 弹窗打开期间监听 Token 过期时间：到期前 2 分钟显示黄色提示条；过期后显示红色提示条，**弹窗保持打开**，用户可继续编辑，提交时若 Token 已失效会提示重新获取。
- 清理弹窗关闭时的 Token 监听定时器，避免泄漏。

### 5.3 业务接入建议

- **禁止**按 `expireSeconds` 自动 `destroy()` 或重建弹窗。
- 建议在 Token 剩余不足 5 分钟时，由业务后端重新调用 `launch-token` 并执行 `TicketSDK.refreshLaunchToken(newToken)`。

## 六、验收标准

1. 插件弹窗打开后静置 10 分钟，不手动点击关闭时弹窗仍存在。
2. LaunchToken 默认有效期为 7200 秒。
3. 调用 `refreshLaunchToken` 后，上传图片与提交工单可继续使用。
4. Token 过期时弹窗不自动关闭，界面有明确提示文案。

## 七、回滚策略

- 将 `plugin.launch-token-expire-seconds` 改回 `300` 即可恢复旧行为。
- SDK 静态文件回滚至上一版 `ticket-sdk.min.js`。
