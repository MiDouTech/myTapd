# 「待出简报工单」列表口径（my_brief_todo）

## 变更说明（与实现一致）

- **主处理人**：`ticket.assignee_id` 为当前登录用户。
- **工单或简报条件（满足其一即可）**：
  - 工单状态为 **已完成**（`completed`）；或
  - 该工单在 `bug_report_ticket` 上关联至少一条未删除的 `bug_report`，且简报状态为 **已退回**（`REJECTED`，产品文案「已退回」）。
- **排除**：
  - 存在任一关联简报状态为 **已归档**（`ARCHIVED`）；
  - 存在任一关联简报状态为 **待审核**（`PENDING_REVIEW`）。

## 同步位置

- 工单分页：`ticket-platform/.../mapper/ticket/TicketMapper.xml` 中 `view == 'my_brief_todo'`。
- 日报「待简报」工单：`DailyReportMapper.xml` 中 `selectPendingVerifyTickets` 与上述「已完成 / 已退回简报」分支口径一致。
