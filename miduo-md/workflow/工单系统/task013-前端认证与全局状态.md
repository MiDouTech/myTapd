# Task013：前端认证与全局状态

> **业务模块**：工单系统（前端）  
> **依赖**：Task011、Task003（后端企微认证就绪）  
> **预估工时**：2天  
> **对应产品文档**：5.1 登录页、4.6.1 企微扫码登录、认证与 JWT 管理

---

## 一、任务目标

实现企业微信扫码登录页（产品 5.1）、前端 Token 与用户信息管理、路由鉴权；封装 axios 请求（baseURL、鉴权头、401 处理），建立 Pinia 用户态与登录态，为后续所有需登录接口提供统一鉴权基础。

## 二、交付物清单

| 序号 | 交付物 | 路径/说明 |
|------|--------|-----------|
| 1 | 登录页 | 企微扫码登录 UI（二维码区域、文案“请使用企业微信扫码登录”），调用后端企微 OAuth 相关接口获取二维码与轮询结果 |
| 2 | 登录流程 | 扫码成功 → 后端回调/轮询拿到 token → 前端存储 token、跳转首页 |
| 3 | Token 存储 | 安全存储（如 sessionStorage/localStorage，按安全规范），请求头携带（如 Authorization: Bearer xxx） |
| 4 | 用户信息 | 登录后获取用户信息接口调用，Pinia store 存储 user（id、name、avatar 等） |
| 5 | axios 封装 | 请求/响应拦截器：自动带 token、401 统一跳转登录、错误提示 |
| 6 | 路由守卫 | 未登录访问需登录页时重定向到 /login，登录后重定向回原目标或首页 |
| 7 | 登出 | 清除 token 与用户态、跳转登录页 |

## 三、与产品文档对应

- **5.1 登录页**：仅保留企微扫码登录，无需账号密码  
- **4.6.1**：企微 OAuth2.0 扫码登录，用户自动绑定  

## 四、验收标准

- [ ] 未登录访问受保护路由会跳转登录页  
- [ ] 扫码登录成功后进入首页，请求头带 token，用户信息展示正确（如头像、昵称）  
- [ ] 401 时自动跳转登录并清除本地登录态  
- [ ] 登出后无法访问受保护路由  

## 五、产出说明

本 Task 完成后，前端具备完整登录与鉴权能力，Task004 工单接口与后续所有需登录接口均可依赖本 Task 的 axios 与路由守卫。
