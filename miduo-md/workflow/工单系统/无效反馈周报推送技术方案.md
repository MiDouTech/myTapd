# 无效反馈月报并入日报管理技术方案

## 1. 方案概要

在现有“日报管理”能力基础上，将“无效反馈周报”改造为“无效反馈月报”，并并入日报管理页面，复用：

1. system_config 配置模型；
2. 企微 webhook 发送器；
3. 日报定时任务 + Redis 分钟级去重机制。

---

## 2. 影响范围

## 2.1 后端

- `ticket-application`：调整无效反馈报表应用服务（月度统计 + 复用日报配置）；
- `ticket-controller`：保留无效反馈报表预览/手动推送接口，配置接口降级为兼容；
- `ticket-job`：移除独立周报任务，改由日报任务同刻触发；
- `ticket-infrastructure`：保留无效反馈统计 SQL，时间口径改为按月；
- `ticket-common`：沿用日报去重，不再依赖独立周报调度去重。

## 2.2 前端

- 管理端移除“无效反馈周报”独立菜单；
- 在“日报管理”页面中并入无效反馈月报预览/手动推送区块；
- 保留无效反馈报表 API 封装和类型定义供日报管理页调用。

## 2.3 文档

- 新增需求文档与技术方案文档；
- 更新接口编号映射文档；
- 更新 `readme.md` 功能说明。

---

## 3. 关键设计决策

1. **口径复用现有字段**：无效反馈直接使用 `manual_valid_report = NO`，不新增判定字段；
2. **统计按月窗口**：当月 1 日 00:00:00 到下月 1 日 00:00:00；
3. **反馈人口径固定**：使用 `ticket.creator_id` 映射用户姓名；
4. **推送时间统一**：无效反馈月报和日报由同一调度触发；
5. **管理入口统一**：无效反馈月报界面并入日报管理，减少配置分叉。

---

## 4. 数据与SQL设计

新增三类查询：

1. `selectWeeklyInvalidFeedbackTotal`：统计总数；
2. `selectWeeklyInvalidFeedbackByReporter`：按反馈人统计数量；
3. `selectWeeklyInvalidFeedbackDetail`：查询明细列表（按关闭时间倒序）。

查询条件统一包含：

- `ticket.deleted = 0`；
- `ticket_bug_info.deleted = 0`；
- `ticket.status = 'closed'`；
- `ticket_bug_info.manual_valid_report = 'NO'`；
- 关闭时间在月范围内；
- 分类 ID 过滤（可选）。

---

## 5. 配置与调度

无效反馈月报统一复用 `DAILY_REPORT` 配置：

- `daily_report_enabled`
- `daily_report_cron`
- `daily_report_webhook_urls`
- `daily_report_stat_category_ids`

调度策略：

- 由 `DailyReportJobHandler` 统一触发；
- 日报推送完成后，同一轮执行无效反馈月报推送；
- 复用日报分钟级去重锁，避免重复触发。

## 7. 风险与回滚

## 7.1 风险

1. 月报明细过长触发企微单条消息长度限制；
2. 分类 ID 配置错误导致“统计为 0”；
3. 历史数据无关闭时间导致月报漏统。

## 7.2 缓解措施

1. 增加明细上限与消息长度保护；
2. 在日报管理页明确提示“按分类 ID 生效”；
3. 关闭时间为空时兜底使用解决时间。

## 7.3 回滚策略

1. 配置层回滚：关闭 `daily_report_enabled`（日报与月报同步停推）；
2. 代码层回滚：恢复无效反馈独立页面和独立调度逻辑。
