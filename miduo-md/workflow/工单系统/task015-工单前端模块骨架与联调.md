# Task015：工单前端模块骨架与联调

> **业务模块**：工单系统（前端）  
> **依赖**：Task011、Task012、Task013、Task014、Task004（后端工单核心接口就绪）  
> **预估工时**：3天  
> **对应产品文档**：4.2 工单管理、4.3 分类与模板管理；Task004 交付物「前端页面」

---

## 一、任务目标

完成工单相关前端页面骨架与后端联调：工单列表（多视图、筛选、排序、分页）、工单详情、创建工单（选择分类→模板→填写→提交）、分类管理（树形）；对接 Task004 提供的接口，保证前后端联调通过、数据真实展示。

## 二、交付物清单

| 序号 | 交付物 | 路径/说明 |
|------|--------|-----------|
| 1 | 工单列表页 | 我创建的/我待办的/我参与的/我关注的/所有工单 视图切换；筛选（编号、标题、分类、状态、优先级、创建人、处理人、时间范围、SLA）；排序、分页；使用 Task014 表格与分页组件 |
| 2 | 工单详情页 | 基本信息、描述、自定义字段、附件、评论；操作按钮：处理、转派、关闭、关注；与 GET /api/ticket/detail/{id} 等接口联调 |
| 3 | 创建工单页 | 选择分类 → 加载模板 → 动态表单 → 提交；与 POST /api/ticket/create、分类树、模板列表接口联调 |
| 4 | 分类管理页 | 三级分类树展示与维护（增删改），与分类 CRUD 接口联调 |
| 5 | API 层 | api/ticket、api/category、api/template 等封装，接口编号在注释中标注（与功能接口对应关系一致） |
| 6 | 类型定义 | 工单、分类、模板等 TypeScript 类型与后端 DTO 对齐 |
| 7 | 错误与空态 | 接口失败、无数据时使用 Task014 空状态与消息反馈 |

## 三、接口对应（与 Task004 一致）

| 接口     | 方法 | 路径                     | 前端用途         |
|----------|------|--------------------------|------------------|
| 分类树   | GET  | /api/category/tree       | 侧栏/创建工单/分类管理 |
| 分类详情 | GET  | /api/category/detail/{id}| 分类编辑         |
| 分类创建 | POST | /api/category/create     | 分类管理         |
| 分类更新 | PUT  | /api/category/update/{id}| 分类管理         |
| 分类删除 | DELETE | /api/category/delete/{id}| 分类管理         |
| 模板列表 | GET  | /api/template/list       | 创建工单         |
| 工单创建 | POST | /api/ticket/create       | 创建工单         |
| 工单分页 | GET  | /api/ticket/page         | 工单列表         |
| 工单详情 | GET  | /api/ticket/detail/{id}  | 工单详情         |
| 工单分派 | PUT  | /api/ticket/assign/{id}  | 工单详情/列表    |
| 工单处理 | PUT  | /api/ticket/process/{id} | 工单详情         |
| 工单关闭 | PUT  | /api/ticket/close/{id}   | 工单详情         |

## 四、验收标准

- [ ] 工单列表多视图、筛选、排序、分页与后端数据一致，无 mock 数据  
- [ ] 工单详情可打开并展示完整信息，处理/转派/关闭可调通后端  
- [ ] 创建工单流程：选分类→选模板→填表单→提交，工单可真实创建并出现在列表  
- [ ] 分类管理树形展示与 CRUD 与后端一致  
- [ ] 所有请求经 axios 封装，鉴权正常，401 处理符合 Task013  
- [ ] 前端可编译、可启动，与后端联调无阻塞错误  

## 五、产出说明

本 Task 完成后，工单系统前端核心流程（列表、详情、创建、分类管理）与后端 Task004 闭环，可作为第一期 MVP 前端交付基础；后续 Task005~010 的后端能力可在本骨架基础上扩展页面与接口。
