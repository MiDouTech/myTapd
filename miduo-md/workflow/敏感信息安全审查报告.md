## 修复详情

### 1. 开发登录密码外部化（高危）

**问题**：`AuthApplicationService.java` 中硬编码了三个常量：
- `DEV_USERNAME = "admin"`
- `DEV_PASSWORD = "admin2026"`
- `PHONE_LOGIN_PASSWORD = "admin123"`

**修复**：
- 将三个常量改为 `@Value` 注入的字段
- 在 `application-dev.yml` 中添加配置项（密码值由环境变量或 Nacos 提供）
- 在 Nacos secrets 模板中添加对应占位符

### 2. Docker Compose 密码外部化（高危）

**问题**：`docker-compose.yml` 中 MySQL 和 MinIO 密码写死。

**修复**：
- 改为从环境变量读取（使用 `${VAR:?error_msg}` 语法强制要求）
- 新增 `.env.example` 模板文件供开发者参考
- `.gitignore` 中确保 `.env` 文件被忽略

### 3. 内网地址脱敏（中危）

**问题**：`bootstrap.yml`、`README.md`、文档中泄露了内网 Nacos 地址 `10.0.4.4:8848` 和命名空间 UUID。

**修复**：
- `bootstrap.yml` 默认值改为 `localhost:8848`，命名空间默认值清空
- 文档中的具体地址替换为环境变量引用说明

### 4. .gitignore 增强（防御性）

**新增忽略规则**：
- `.env` / `.env.local` / `.env.*.local` 等环境文件
- `deployment/docker/.env`（ticket-platform 目录）

## 不构成泄露风险的条目（已评估，无需处理）

| 条目 | 原因 |
|------|------|
| `miduo-md/nacos-config/ticket-platform-secrets-*.yml` | 全部为 `<YOUR_*>` 占位符，不含真实密钥 |
| `miduo-frontend/.env.development` / `.env.production` | 仅含 `VITE_*` 前端配置变量，无密钥 |
| `.env.prod.example` / `.env.test.example` | 模板文件，密码为 `change_me` 占位 |
| `application-dev.yml` | 不含数据库密码/密钥，敏感值由 Nacos 注入 |
| `.github/workflows/cd.yml` | 引用 `${{ secrets.* }}` GitHub Secrets，非明文 |
| API 设计文档中的 `eyJhbGci...` | 截断的示意性 JWT，非完整可用 token |
| `QiniuUploadService.java` 等代码 | AK/SK 从配置读取，无硬编码明文 |

## 建议的后续行动

### 短期（发布开源前必须完成）

1. **Git 历史清理**：本次修复仅针对当前文件内容。如果仓库即将开源，建议使用 `git filter-repo` 或 `BFG Repo-Cleaner` 清理 Git 历史中的敏感信息残留（如历史提交中的 `root123456`、`admin2026`、完整 JWT token 等）
2. **轮换所有凭证**：假设当前所有被暴露的凭证（MySQL 密码、JWT Secret、开发登录密码等）已被泄露，建议在所有环境中轮换

### 中期

3. **CI/CD 集成密钥扫描**：在 GitHub Actions 中集成 [gitleaks](https://github.com/gitleaks/gitleaks) 或 [truffleHog](https://github.com/trufflesecurity/trufflehog)，对每个 PR 自动扫描
4. **pre-commit hook**：配置 `pre-commit` 钩子，在本地提交前检测敏感信息

### 长期

5. **密钥管理**：评估使用 HashiCorp Vault 或云厂商 KMS 统一管理密钥
6. **定期审查**：每季度执行一次全仓库敏感信息扫描
