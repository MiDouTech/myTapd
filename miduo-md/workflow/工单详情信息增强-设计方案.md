# 工单详情信息增强 — 技术设计方案

> 版本：v1.0  
> 日期：2026-03-31  
> 对应 PRD：`miduo-md/business/工单详情信息增强PRD.md`

## 1. 范围摘要

- **P0 + P1（本期实现）**：`ticket.resolution_summary` 处理结论；终态流转/关闭时写入；公开详情返回处理结论、系统「处理动态」、附件列表、描述与问题描述去重标记；缺陷工作流客服信息扩展排障字段（存 `ticket_bug_info`，URL 服务端脱敏）。
- **安全**：公开接口不返回内部 JSON 变更日志；`ticket_log` 仅白名单动作（CREATE、ASSIGN、TRANSFER、TRANSIT、RETURN）映射为摘要动态。

## 2. 数据库

- Flyway：`V41__ticket_resolution_and_bug_troubleshooting.sql`
  - `ticket.resolution_summary` varchar(2000)
  - `ticket_bug_info`：`troubleshoot_*` 六列

## 3. 后端要点

- `TicketWorkflowAppService.transit`：目标为工作流终态时合并写入 `resolutionSummary`（`TransitInput` 新字段，优先于 remark）。
- `TicketApplicationService.buildPublicDetailOutput`：组装 `activities`、`publicAttachments`、`descriptionDuplicateOfProblemDesc`、缺陷排障子对象（URL 再次 `PublicUrlSanitizer`）。
- `TicketBugApplicationService`：客服信息更新时排障字段可选；未传排障字段时保留库内原值；新建机器人初始化不受影响。
- 工具类：`ticket-common` — `PublicUrlSanitizer`、`TroubleshootClientType` 枚举。

## 4. 前端要点

- `TicketPublicView.vue`：区块顺序对齐 PRD；时间线 = `activities` + `comments` 按时间合并。
- `TicketDetailView.vue`：缺陷工作流客服 Tab 增加排障表单；关闭/终态流转增加处理结论；描述与问题描述重复时折叠描述区。
- `BugDetailInfoPanel.vue`：展示 `resolutionSummary`。

## 5. 接口契约（向后兼容）

以下为**字段扩展**，路径与既有编号不变：

| 接口 | 编号 | 变更 |
|------|------|------|
| GET `/api/ticket/detail/{id}` | API000008 | 增加 `resolutionSummary`；`bugCustomerInfo` 增加排障字段 |
| PUT `/api/ticket/close/{id}` | API000011 | 请求体增加可选 `resolutionSummary` |
| PUT `/api/ticket/process/{id}` | API000010 | 请求体增加可选 `resolutionSummary`（透传流转） |
| GET `/api/ticket/{id}/available-actions` | API000014 | `actions[]` 增加 `targetTerminal` |
| PUT `/api/ticket/transit/{id}` | API000015 | 请求体增加可选 `resolutionSummary` |
| PUT `/api/ticket/bug/customer-info/{id}` | API000021 | 请求体增加可选排障六字段 |
| GET `/api/open/ticket/{ticketNo}` | API000417 | 增加 `resolutionSummary`、`activities`、`publicAttachments`、`descriptionDuplicateOfProblemDesc`；`bugCustomerInfo.troubleshooting` |

## 6. 回滚

- 回滚迁移需单独脚本删除列（生产慎用）；代码回滚后新列可保留为空不影响读路径。
