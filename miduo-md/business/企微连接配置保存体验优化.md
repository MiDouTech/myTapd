# 企微连接配置保存体验优化

## 背景

管理端「企微连接配置」保存时，此前要求每次保存都完整填写应用 Secret、回调 AESKey 等敏感字段，运维仅调整非敏感项时成本高。

## 目标

1. 详情接口返回的敏感信息保持脱敏展示（前若干位 + `****` + 后若干位），回调 Token 与 Secret、AESKey 一致，不落库明文到接口响应。
2. 更新配置时，Secret、AESKey、回调 Token 等字段**留空表示不修改**，无需重复输入完整值。
3. 当本次提交解析后的有效配置与数据库当前记录完全一致时，**不执行 UPDATE**，避免无意义的写库与 token 缓存刷新。

## 非目标

- 不改变企微连接、组织同步、回调验签等运行时行为。
- 不新增对外 API 路径，仅调整 `API000422` 响应字段与 `API000423` 入参语义。

## 验收

1. 已存在配置时，仅修改 CorpID/超时等非密钥字段并保存，数据库 `update_time` 无变化（或等价于未触发更新，依实现为准）。
2. 已存在配置时，Secret/AESKey/回调 Token 输入框留空保存，库中对应密文字段不变。
3. `GET /api/wecom/config/detail` 中不再返回明文 `callbackToken`，仅返回 `callbackTokenMasked`。
