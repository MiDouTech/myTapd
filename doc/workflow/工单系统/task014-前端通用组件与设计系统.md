# Task014：前端通用组件与设计系统

> **业务模块**：工单系统（前端）  
> **依赖**：Task011  
> **预估工时**：2天  
> **对应产品文档**：5.2、5.4 设计原则；`.cursor/rules/frontend-ui-standards.mdc`

---

## 一、任务目标

建立工单前端与产品/规范一致的设计系统：表格、分页、主色调、空状态、加载态、消息提示等通用组件或封装，确保后续工单列表、详情、看板等页面风格统一且符合规范。

## 二、交付物清单

| 序号 | 交付物 | 路径/说明 |
|------|--------|-----------|
| 1 | 表格规范组件/封装 | 无边框、斑马纹、表头 #f5f7fa、居中对齐、支持排序与多选（按需）、操作列固定右侧，符合 frontend-ui-standards |
| 2 | 分页组件封装 | 10/20/50/100 条，默认 20，layout 含 total/sizes/prev/pager/next/jumper，background |
| 3 | 主题与变量 | 主色 #1675d1、表头背景 #f5f7fa、悬停 #f0f9ff、字体 14px、表头 font-weight 500 |
| 4 | 空状态组件 | 无数据时的统一展示（文案、图标），便于列表/看板复用 |
| 5 | 加载态 | 表格 loading、按钮 loading、全局或局部 loading 使用方式统一 |
| 6 | 消息/反馈 | 与 Element Plus 的 Message/MessageBox 使用约定（成功/失败/确认），错误码与文案映射预留 |
| 7 | 文档或说明 | 通用组件使用说明（Props、插槽、示例），便于 Task015 及后续页面直接引用 |

## 三、与规范对应

- **frontend-ui-standards.mdc**：分页、表格、主色调、响应式（表格横向滚动、操作列固定）  
- **5.4 设计原则**：简洁高效、信息层次分明、视觉反馈（优先级/SLA 颜色）、快捷操作  

## 四、验收标准

- [ ] 表格与分页在示例页中展示符合规范，主色调与表头/悬停色正确  
- [ ] 空状态、加载态在示例中可演示  
- [ ] 组件嵌套深度、props 透传符合项目红线（≤2 层）  
- [ ] 后续工单列表可直接复用表格+分页封装，无需重复写样式  

## 五、产出说明

本 Task 完成后，工单前端具备统一的设计语言与可复用组件，Task015 工单列表/详情/创建页可基于此快速实现并保持风格一致。
